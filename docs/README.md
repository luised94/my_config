# docs/

Reference documentation for personal configuration and workflows.

## Naming

- **Extension:** `.md` (use `.qmd` only when Quarto features are needed)
- **Convention:** lowercase, underscores separate fields, hyphens within multi-word fields
- **Examples:** `git_workflow.md`, `python_development-setup.md`
- **No numbering** - order doesn't matter
- **No subdirectories** unless file count exceeds ~10-12 topic files

## Scope

Each file covers one topic and should be self-contained. A reader shouldn't need to read other docs as prerequisites.
